Detailed Description

A simple demo class to implement the operation of an abstract biological neuron. The operating principle of neuronal processing, and programming details:

The class adjusts member scGenComp_PU_Bio::m_Membrane_V (the membrane potential). Actually, it implements the AP creation as shown in Figure fig_AP_Conceptual.

Given that the membrane's potential rather than time of instruction count defines processing phases, method scGenComp_PU_Abstract::Heartbeat_method() is called periodically, with HEARTBEAT_TIME_DEFAULT (simulated) time (presently set to \(10\ \mu s\)). That function internally calls one of its state-dependent Heartbeat_XXX_Do() methods.

InputReceived_Do() imitates chargeup: membranes voltage increases promptly by \(6\ mV\) after receiving a synaptic input

Heartbeat_Computing_Do() imitates membrane's discharge by decreasing its voltage by \(1\ mV\)

Heartbeat_Delivering_Do() imitates creating a sawtooth-like AP on the membrane

Heartbeat_Relaxing_Do() imitates restoring potential to RestingMembranePotential by changing its value by by \(\pm 1\ mV\)

The states of the neuronal state machine are controlled by methods Heartbeat_XXX_Stop()

Heartbeat_Computing_Stop() passes to phase 'Delivering' if membrane's voltage exceeds the ThresholdPotential or passes to phase 'Relaxing' if it falls below RestingMembranePotential (calls scGenComp_PU_Abstract::Failed_method() )

Heartbeat_Delivering_Stop() passes to phase "Relaxing", practically at a constant time. It increases membrane's voltage until \(\approx 150\ mV\) reached, then decreases in somewhat lower pace until ThresholdPotential reached (imitate hyperpolarization)

Heartbeat_Relaxing_Stop() decreases membrane potential until -ThresholdPotential reached, then restores RestingMembranePotential reached

The class needs (empty) constructor and destructor. Initialize_Do() is only called in constructor (preserves voltage between spikes)
